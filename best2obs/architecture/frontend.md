# Frontend / User Interfaces

В проекте нет классического web frontend. Пользовательский интерфейс - Telegram-чат.

## Telegram UI

Основной файл: `app/bot/telegram_bot.py`.

Что делает:

- `/start` - приветствие от бота Любовь.
- `/status` - технический статус процесса и БД.
- текстовые сообщения - передаются в `handle_incoming`.
- голосовые сообщения - сначала транскрибируются, затем идут в тот же обработчик.
- typing indicator - отправляется, пока AI/backend обрабатывает запрос.
- фото - отправляются после ответа, одиночной фотографией или media group.

## Media UX

Фото подбираются через `app/services/media_service.py`.

Правила:

- При явном запросе фото ограничения игнорируются.
- Автоматические фото не должны спамить: повторная отправка ограничена окном 12 часов.
- Для беседок фото должны соответствовать доступным и подходящим вариантам.
- После успешной оплаты отправляется фото забронированной услуги.

## Риски интерфейса

- Telegram media group может частично не отправиться при проблемах с файлами или таймаутом.
- Если текст ответа говорит о фото, а media-service не нашел файл, пользователь увидит несоответствие.
- Важно тестировать реальные фото после замены файлов в `app/images/`.
