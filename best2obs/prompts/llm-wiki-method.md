# LLM Wiki Method

Эта страница фиксирует, как использовать `best2obs/` по паттерну LLM Wiki.

## Идея

Обычный RAG каждый раз заново ищет фрагменты в документах. LLM Wiki работает иначе: агент постепенно строит и поддерживает связанную markdown-базу знаний. Новые источники не просто индексируются, а интегрируются в существующие страницы.

Для `best2` это значит:

- важные знания о коде, архитектуре, багам, решениях и тестах должны попадать в `best2obs/`;
- страницы должны ссылаться друг на друга;
- противоречия и устаревшие сведения нужно отмечать и исправлять;
- полезные ответы из чата можно сохранять как новые wiki-страницы.

## Слои

### Raw sources

Источник правды, который агент читает, но не меняет без явного запроса:

- production-код `app/`, `scripts/`, `config/`;
- `README.md`, `PLAN.md`, `information.md`, `.env.example`;
- пользовательские требования из чата;
- логи и результаты тестов;
- внешние документы, если пользователь добавит их в проект.

### Wiki

Markdown-страницы в `best2obs/`. Их агент поддерживает:

- `architecture/` - как устроена система;
- `bugs/` - текущие проблемы и риски;
- `decisions/` - принятые решения;
- `roadmap/` - задачи и проверки;
- `daily/` - дневные заметки, если понадобится;
- `prompts/` - рабочие инструкции и команды.

### Schema

Правила обслуживания wiki:

- `AGENTS.md`;
- `best2obs/prompts/codex-workflow.md`;
- эта страница.

## Операции

### Ingest

Когда появляется новый источник:

1. Прочитать источник.
2. Выделить важные факты, решения, баги, риски и задачи.
3. Обновить релевантные страницы.
4. Если нужно, создать новую страницу.
5. Обновить `index.md`.
6. Добавить запись в `log.md`.

### Query

Когда пользователь спрашивает по проекту:

1. Прочитать `index.md` и `log.md`.
2. Найти релевантные wiki-страницы.
3. При необходимости проверить raw sources.
4. Ответить с учетом накопленной памяти.
5. Если вывод полезен для будущего, сохранить его в wiki и обновить log.

### Lint

Периодическая проверка здоровья wiki:

- найти противоречия между страницами;
- найти устаревшие утверждения;
- найти страницы без ссылок;
- найти важные темы без отдельной страницы;
- обновить roadmap и bugs;
- добавить запись в log.

## Index и log

`index.md` - карта содержания wiki. Обновлять при создании или важном изменении страниц.

`log.md` - хронология. Записи добавлять сверху вниз или в порядке времени, с датой и коротким смыслом.

## Важное правило

Не оставлять важные знания только в чате. Если информация поможет будущей работе над ботом, она должна попасть в `best2obs/`.
