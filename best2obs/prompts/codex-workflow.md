# Codex Workflow Prompts

Короткие команды для дальнейшей работы:

- `Перед задачей прочитай best2obs/index.md и best2obs/log.md.`
- `Проверь auth flow и обнови best2obs.`
- `Исправь баг и запиши изменения в best2obs/log.md.`
- `Посмотри roadmap/pre-launch.md и предложи следующий шаг.`
- `После правки обнови architecture и current-known-issues.`
- `Проведи аудит post-booking flow и запиши найденные риски в bugs/current-known-issues.md.`
- `Проверь переносы броней по roadmap/pre-launch.md и обнови log.md.`
- `Сравни текущее поведение с архитектурой и поправь best2obs, production-код не трогай.`
- `Создай decision note в best2obs/decisions/ по выбранному подходу.`
- `После тестов добавь краткий итог в best2obs/log.md.`
- `Ingest новый источник в best2obs по LLM Wiki workflow.`
- `Сделай lint best2obs: найди противоречия, устаревшие страницы и недостающие ссылки.`
- `Ответь на вопрос по wiki, а полезный вывод сохрани новой страницей.`
- `Обнови index.md и log.md после ingest/query/lint.`

## Рабочий порядок

1. Читать `best2obs/index.md`.
2. Читать `best2obs/log.md`.
3. Смотреть релевантные страницы architecture/bugs/roadmap.
4. Если задача про знания, выбрать режим: Ingest, Query или Lint.
5. Делать задачу.
6. Обновлять память проекта.
