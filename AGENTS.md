# AGENTS.md

Правила работы с проектом `best2`:

- Перед задачами читать `best2obs/index.md` и `best2obs/log.md`.
- Использовать `best2obs/` как долговременную память проекта.
- После значимых изменений обновлять `best2obs/log.md`.
- Если найдены баги, фиксировать их в `best2obs/bugs/`.
- Если изменена архитектура, обновлять `best2obs/architecture/`.
- Если принято решение, записывать его в `best2obs/decisions/`.
- Если появились задачи, обновлять `best2obs/roadmap/`.
- Не менять production-код без отдельного запроса.

## Graphify workflow

- `best2graph/` содержит Graphify-карту кода.
- `best2obs/` остается памятью решений проекта; Graphify ее не заменяет.
- `best2info/` остается клиентской базой знаний.
- Для архитектуры, рефакторинга и поиска места правки сначала использовать Graphify, затем открывать конкретные файлы.
- Не запускать `graphify codex install` без отдельного разрешения.
- После значимых изменений кода обновлять карту: `.\best2graph\update_graph.ps1`.
- Запрос к карте: `.\best2graph\.venv\Scripts\graphify.exe query "QUESTION" --graph .\best2graph\graphify-out\graph.json --budget 1200`.

## LLM Wiki workflow

`best2obs/` ведется как LLM Wiki: агент не просто отвечает в чат, а поддерживает постоянную, связанную базу знаний проекта.

Слои памяти:

- Raw sources: код проекта, README, PLAN, `.env.example`, конфиги, логи, пользовательские сообщения и внешние документы. Их читать как источник правды, не переписывать без запроса.
- Wiki: markdown-страницы в `best2obs/`. Их агент может создавать и обновлять, чтобы знания накапливались.
- Schema: `AGENTS.md` и `best2obs/prompts/codex-workflow.md`. Здесь описаны правила работы с wiki.

Операции:

- Ingest: при добавлении нового источника прочитать его, извлечь важное, обновить релевантные страницы, `index.md` и `log.md`.
- Query: при вопросе сначала искать по `index.md`, затем читать нужные страницы wiki и только потом код/сырьевые источники.
- Lint: периодически проверять wiki на противоречия, устаревшие утверждения, orphan pages, недостающие ссылки и новые задачи.

Правило накопления:

- Важные выводы из обсуждений не должны оставаться только в чате. Если они полезны для будущей работы, оформить их в `best2obs/`.
